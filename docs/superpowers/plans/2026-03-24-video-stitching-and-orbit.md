# Video Stitching & Orbit Video Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add video stitching and 360° orbit video tools to Sogni Chat, powered by a reusable pipeline framework.

**Architecture:** Port the manual MP4 concatenation engine from sogni-photobooth (no external dependency — pure DataView/Uint8Array parsing). Build a `ToolPipeline` abstraction that chains tool executions with intermediate state. Create `stitch_video` (standalone concatenation) and `orbit_video` (composite 3-step pipeline) tools.

**Tech Stack:** TypeScript, manual MP4 container manipulation, existing tool registry pattern, IndexedDB gallery persistence.

**Spec:** `docs/superpowers/specs/2026-03-24-video-stitching-and-orbit-design.md`

**Spec correction:** The spec mentions `mp4box` as a dependency, but the photobooth implementation (`../sogni-photobooth/src/utils/videoConcatenation.js`, 2,276 lines) uses manual MP4 parsing with DataView/Uint8Array — no external library. The port follows the same approach. No `mp4box` dependency is needed. Skip the `package.json` change mentioned in the spec.

const S3_DOWNLOAD_DELAY_MS = 150;

async function downloadVideos(
  urls: string[],
  onProgress?: (progress: number) => void,
): Promise<ArrayBuffer[]> {
  const buffers: ArrayBuffer[] = [];
  for (let i = 0; i < urls.length; i++) {
    if (i > 0) await new Promise(r => setTimeout(r, S3_DOWNLOAD_DELAY_MS));
    const response = await fetch(urls[i]);
    if (!response.ok) throw new Error(`Failed to download video ${i}: ${response.status}`);
    buffers.push(await response.arrayBuffer());
    onProgress?.((i + 1) / urls.length * 0.5); // First 50% is download
  }
  return buffers;
}
```

Add the main export that ties everything together:

```typescript
/**
 * Concatenate multiple MP4 video files into a single continuous video.
 *
 * Uses lossless container manipulation — no re-encoding.
 * Handles H.264 B-frames (ctts), edit lists (edts) for iOS/QuickTime
 * compatibility, and proper chunk offset recalculation.
 *
 * @param videoUrls - Array of video URLs to concatenate in order
 * @param onProgress - Optional progress callback (0-1)
 * @returns Blob containing the concatenated MP4
 */
export async function concatenateVideos(
  videoUrls: string[],
  onProgress?: (progress: number) => void,
): Promise<Blob> {
  if (videoUrls.length === 0) throw new Error('No videos to concatenate');
  if (videoUrls.length === 1) {
    // Single video — just download and return
    const response = await fetch(videoUrls[0]);
    if (!response.ok) throw new Error(`Failed to download video: ${response.status}`);
    return new Blob([await response.arrayBuffer()], { type: 'video/mp4' });
  }

  // Download all videos with rate limiting
  const buffers = await downloadVideos(videoUrls, onProgress);

  // Concatenate MP4s
  onProgress?.(0.6);
  const result = await concatenateMP4s_Base(buffers);
  onProgress?.(0.9);

  return new Blob([result], { type: 'video/mp4' });
}
```

Create `src/tools/shared/pipeline.ts` with the exported interfaces from the spec:

```typescript
/**
 * ToolPipeline — a reusable framework for multi-step tool orchestration.
 *
 * Chains tool executions with intermediate state, phased progress reporting,
 * and graceful error handling. Designed for composite tools that need to
 * orchestrate multiple sub-tools sequentially (e.g., orbit_video).
 *
 * ## Concepts
 *
 * - **PipelineStep**: A unit of work — runs a tool N times or a custom function.
 * - **PipelineState**: Accumulated state flowing between steps (URLs, data).
 * - **StepResult**: Output from a single tool invocation within a step.
 * - **PipelineConfig**: The full pipeline definition (steps + initial state).
 *
 * ## Execution Model
 *
 * - Steps execute sequentially (step N must complete before step N+1 starts)
 * - Invocations within a step execute sequentially (one SDK job at a time)
 * - When `customExecute` is provided, the `count` field is ignored
 * - Sub-tool progress is intercepted and re-emitted with phased labels
 * - AbortSignal is respected — pipeline stops after current invocation
 *
 * ## Progress Mapping
 *
 * Sub-tool callbacks are wrapped to report phased progress:
 * - stepLabel: "{step.label} {i+1}/{step.count}"
 * - perJobProgress: per-invocation slots within the step
 * - resultUrls/videoResultUrls: accumulated as sub-tools complete
 *
 * ## Usage Example
 *
 * ```typescript
 * const config: PipelineConfig = {
 *   initialState: { imageUrls: [], videoUrls: [], data: {} },
 *   steps: [
 *     {
 *       label: 'Generating angles',
 *       toolName: 'change_angle',
 *       count: 4,
 *       buildArgs: (state, i) => ({ description: ANGLES[i] }),
 *       collectResults: (state, results) => ({
 *         ...state,
 *         imageUrls: results.map(r => r.imageUrls[0]),
 *       }),
 *     },
 *     // ... more steps
 *   ],
 * };
 *
 * const finalState = await executePipeline(config, context, callbacks);
 * ```
 */

import type { ToolExecutionContext, ToolCallbacks, ToolName } from '../types';

export interface PipelineState {
  /** Image URLs accumulated across steps */
  imageUrls: string[];
  /** Video URLs accumulated across steps */
  videoUrls: string[];
  /** Arbitrary data passed between steps (e.g., stitchedUrl, angleStartIndex) */
  data: Record<string, unknown>;
}

export interface StepResult {
  /** Raw JSON string returned by the tool handler */
  rawResult: string;
  /** Image URLs produced by this invocation */
  imageUrls: string[];
  /** Video URLs produced by this invocation */
  videoUrls: string[];
}

export interface PipelineStep {
  /** Human-readable label shown in stepLabel (e.g., "Generating angle views") */
  label: string;
  /** Tool name to execute via registry. Null for custom-execute steps. */
  toolName: ToolName | null;
  /** Number of sequential invocations. Ignored when customExecute is provided. */
  count: number;
  /** Derive tool args for the i-th invocation from current state. */
  buildArgs: (state: PipelineState, index: number) => Record<string, unknown>;
  /** Custom execution (e.g., stitching). When provided, toolName and count are ignored. */
  customExecute?: (
    state: PipelineState,
    context: ToolExecutionContext,
    callbacks: ToolCallbacks,
  ) => Promise<StepResult[]>;
  /** Update state after all invocations complete. */
  collectResults: (state: PipelineState, results: StepResult[]) => PipelineState;
}

export interface PipelineConfig {
  /** Parent tool name — used in progress emissions so the UI labels are correct */
  parentToolName: ToolName;
  steps: PipelineStep[];
  initialState: PipelineState;
}
```

- [ ] **Step 2: Write the `executePipeline` function**

Add the main executor below the interfaces:

```typescript
import { toolRegistry } from '../registry';

/**
 * Execute a pipeline: iterate steps sequentially, run sub-tools via the
 * registry, intercept callbacks for phased progress, and flow state
 * between steps.
 *
 * @param config - Pipeline configuration (steps + initial state)
 * @param context - Tool execution context (shared across all sub-tools)
 * @param callbacks - Parent tool's callbacks (progress re-mapped by pipeline)
 * @returns Final pipeline state after all steps complete
 */
export async function executePipeline(
  config: PipelineConfig,
  context: ToolExecutionContext,
  callbacks: ToolCallbacks,
): Promise<PipelineState> {
  let state = { ...config.initialState };

  for (const step of config.steps) {
    // Check abort before starting each step
    if (context.signal?.aborted) {
      console.log(`[PIPELINE] Aborted before step: ${step.label}`);
      break;
    }

    console.log(`[PIPELINE] Starting step: ${step.label}`);
    const stepResults: StepResult[] = [];

    if (step.customExecute) {
      // Custom execution — count is ignored, call once
      callbacks.onToolProgress({
        type: 'progress',
        toolName: config.parentToolName,
        stepLabel: step.label,
      });
      const results = await step.customExecute(state, context, callbacks);
      stepResults.push(...results);
    } else {
      // Tool-based execution — invoke N times sequentially
      for (let i = 0; i < step.count; i++) {
        if (context.signal?.aborted) {
          console.log(`[PIPELINE] Aborted during step "${step.label}" at invocation ${i}`);
          break;
        }

        const args = step.buildArgs(state, i);
        const stepLabel = `${step.label} ${i + 1}/${step.count}`;

        // Wrap callbacks to re-map progress with step context
        const wrappedCallbacks: ToolCallbacks = {
          onToolProgress: (progress) => {
            callbacks.onToolProgress({
              ...progress,
              stepLabel,
            });
          },
          onToolComplete: (toolName, resultUrls, videoResultUrls) => {
            // Collect result URLs but DON'T forward onToolComplete to parent —
            // the pipeline manages completion at the end.
            stepResults.push({
              rawResult: '',
              imageUrls: resultUrls || [],
              videoUrls: videoResultUrls || [],
            });
          },
          onInsufficientCredits: callbacks.onInsufficientCredits,
          onGallerySaved: callbacks.onGallerySaved,
        };

        console.log(`[PIPELINE] Executing ${step.toolName} (${i + 1}/${step.count})`);
        const rawResult = await toolRegistry.execute(
          step.toolName!,
          args,
          context,
          wrappedCallbacks,
        );

        // If the tool returned an error, check if we should abort
        try {
          const parsed = JSON.parse(rawResult);
          if (parsed.error) {
            console.error(`[PIPELINE] Sub-tool error in "${step.label}" invocation ${i}:`, parsed.error);
            // For result-bearing errors (like insufficient credits), stop the pipeline
            if (parsed.error === 'insufficient_credits' || parsed.error === 'no_image') {
              throw new Error(parsed.error);
            }
          }
        } catch (e) {
          if (e instanceof SyntaxError) {
            // Not JSON — treat as success (some tools return non-JSON)
          } else {
            throw e;
          }
        }

        // If wrappedCallbacks.onToolComplete wasn't called (tool returned result
        // directly without calling onToolComplete), parse the raw result
        if (stepResults.length <= i) {
          stepResults.push({
            rawResult,
            imageUrls: [],
            videoUrls: [],
          });
        } else {
          // Update the rawResult on the already-pushed entry
          stepResults[i].rawResult = rawResult;
        }
      }
    }

    // Collect results and update state
    state = step.collectResults(state, stepResults);
    console.log(`[PIPELINE] Completed step: ${step.label} (${stepResults.length} results)`);
  }

  return state;
}
```

import type { ToolExecutionContext, ToolCallbacks } from '../types';
import { concatenateVideos } from '@/utils/videoConcatenation';
import { saveVideoToGallery } from '@/services/galleryService';

export async function execute(
  args: Record<string, unknown>,
  context: ToolExecutionContext,
  callbacks: ToolCallbacks,
): Promise<string> {
  // Defensively coerce array items to numbers (registry doesn't recurse into arrays)
  const rawIndices = args.videoIndices as unknown[];
  if (!Array.isArray(rawIndices) || rawIndices.length === 0) {
    return JSON.stringify({ error: 'invalid_args', message: 'videoIndices must be a non-empty array.' });
  }

  const indices = rawIndices.map(Number);
  if (indices.some(isNaN)) {
    return JSON.stringify({ error: 'invalid_args', message: 'All videoIndices must be valid numbers.' });
  }

  // Resolve indices against session video results
  const videoUrls: string[] = [];
  const invalidIndices: number[] = [];
  for (const idx of indices) {
    const url = context.videoResultUrls[idx];
    if (url) {
      videoUrls.push(url);
    } else {
      invalidIndices.push(idx);
    }
  }

  if (invalidIndices.length > 0) {
    return JSON.stringify({
      error: 'invalid_indices',
      message: `Video indices not found: ${invalidIndices.join(', ')}. Available: 0-${context.videoResultUrls.length - 1}.`,
    });
  }

  if (videoUrls.length < 2) {
    return JSON.stringify({
      error: 'too_few_videos',
      message: 'Need at least 2 videos to stitch. Only found ' + videoUrls.length + '.',
    });
  }

  callbacks.onToolProgress({
    type: 'started',
    toolName: 'stitch_video',
    stepLabel: 'Stitching videos',
    totalCount: 1,
  });

  try {
    const blob = await concatenateVideos(videoUrls, (progress) => {
      callbacks.onToolProgress({
        type: 'progress',
        toolName: 'stitch_video',
        progress,
        stepLabel: 'Stitching videos',
      });
    });

    // Create object URL for immediate playback
    const blobUrl = URL.createObjectURL(blob);

    // Persist to gallery (fire-and-forget)
    saveVideoToGallery({ videoBlob: blob })
      .then(({ galleryImageId }) => {
        callbacks.onGallerySaved?.([], [galleryImageId]);
      })
      .catch((err) => {
        console.error('[STITCH] Failed to save stitched video to gallery:', err);
      });

    callbacks.onToolComplete('stitch_video', [], [blobUrl]);

    return JSON.stringify({
      success: true,
      resultCount: 1,
      mediaType: 'video',
      message: `Successfully stitched ${videoUrls.length} videos into one continuous video.`,
    });
  } catch (err) {
    const message = err instanceof Error ? err.message : String(err);
    console.error('[STITCH] Concatenation failed:', message);
    return JSON.stringify({ error: 'stitch_failed', message });
  }
}
```

```typescript
import type { ToolExecutionContext, ToolCallbacks } from '../types';
import type { PipelineConfig, StepResult } from '../shared/pipeline';
import { executePipeline } from '../shared/pipeline';
import { concatenateVideos } from '@/utils/videoConcatenation';
import { saveVideoToGallery } from '@/services/galleryService';
import { preflightCreditCheck, formatCredits } from '../shared';
import { fetchAngleCostEstimate } from '@/services/creditsService';
import { fetchVideoCostEstimate } from '@/services/creditsService';
import { QUALITY_PRESETS } from '@/config/qualityPresets';
import { getVideoModelConfig, calculateVideoDimensions, calculateVideoFrames } from '@/constants/videoSettings';

/** Fixed azimuth sequence for 360° orbit (90° increments) */
const AZIMUTHS = [
  'front view',
  'right side view',
  'back view',
  'left side view',
] as const;

export async function execute(
  args: Record<string, unknown>,
  context: ToolExecutionContext,
  callbacks: ToolCallbacks,
): Promise<string> {
  const elevation = (args.elevation as string) ?? 'eye-level shot';
  const distance = (args.distance as string) ?? 'medium shot';
  const prompt = (args.prompt as string) ?? 'smooth camera pan';
  const sourceImageIndex = args.sourceImageIndex as number | undefined;

  // Validate source image exists
  if (!context.imageData && context.resultUrls.length === 0) {
    return JSON.stringify({ error: 'no_image', message: 'Please upload or generate an image first.' });
  }

  // Effective quality tier (change_angle forces pro → hq)
  const effectiveTier = context.qualityTier === 'pro' ? 'hq' : (context.qualityTier || 'fast');
  const preset = QUALITY_PRESETS[effectiveTier];

  // Pre-flight credit check: 4 angles + 4 videos (3s each)
  const angleCost = await fetchAngleCostEstimate(
    context.sogniClient, preset.model, preset.steps, preset.guidance, context.tokenType,
  );
  const videoConfig = getVideoModelConfig('ltx23');
  const { width: vidW, height: vidH } = calculateVideoDimensions(
    context.width || 1024, context.height || 1024,
    effectiveTier === 'fast' ? 768 : 1088, 'ltx23',
  );
  const frames = calculateVideoFrames(3, 'ltx23'); // 3 second clips
  const videoCost = await fetchVideoCostEstimate(
    context.tokenType, videoConfig.model, vidW, vidH, frames, videoConfig.fps, videoConfig.steps,
  );
  const totalCost = (angleCost * 4) + (videoCost * 4);

  const preflight = preflightCreditCheck(context, totalCost);
  if (!preflight.ok) return preflight.errorJson;

  // Capture the current result URL count so Step 2 can compute correct indices
  const angleStartIndex = context.resultUrls.length;

  callbacks.onToolProgress({
    type: 'started',
    toolName: 'orbit_video',
    stepLabel: 'Starting 360° orbit',
    totalCount: 9, // 4 angles + 4 videos + 1 stitch
    estimatedCost: totalCost,
  });

  // Build pipeline config
  const config: PipelineConfig = {
    parentToolName: 'orbit_video',
    initialState: {
      imageUrls: [],
      videoUrls: [],
      data: { angleStartIndex },
    },
    steps: [
      // Step 1: Generate 4 angle views
      {
        label: 'Generating angle views',
        toolName: 'change_angle',
        count: 4,
        buildArgs: (_state, i) => ({
          description: `${AZIMUTHS[i]} ${elevation} ${distance}`,
          sourceImageIndex,
        }),
        collectResults: (state, results) => ({
          ...state,
          imageUrls: results.map(r => r.imageUrls[0]).filter(Boolean),
        }),
      },
      // Step 2: Generate 4 transition clips
      {
        label: 'Generating transition clips',
        toolName: 'animate_photo',
        count: 4,
        buildArgs: (state, i) => ({
          prompt,
          frameRole: 'both',
          sourceImageIndex: (state.data.angleStartIndex as number) + i,
          endImageIndex: (state.data.angleStartIndex as number) + ((i + 1) % 4),
          duration: 3,
        }),
        collectResults: (state, results) => ({
          ...state,
          videoUrls: results.map(r => r.videoUrls[0]).filter(Boolean),
        }),
      },
      // Step 3: Stitch all clips
      {
        label: 'Stitching final video',
        toolName: null,
        count: 1,
        buildArgs: () => ({}),
        customExecute: async (state, _ctx, cbs) => {
          const blob = await concatenateVideos(state.videoUrls, (progress) => {
            cbs.onToolProgress({
              type: 'progress',
              toolName: 'orbit_video',
              progress,
              stepLabel: 'Stitching final video',
            });
          });

          const blobUrl = URL.createObjectURL(blob);

          // Persist to gallery (fire-and-forget)
          saveVideoToGallery({ videoBlob: blob, prompt: `360° orbit: ${prompt}` })
            .then(({ galleryImageId }) => {
              cbs.onGallerySaved?.([], [galleryImageId]);
            })
            .catch((err) => {
              console.error('[ORBIT] Failed to save stitched video to gallery:', err);
            });

          return [{ rawResult: '', imageUrls: [], videoUrls: [blobUrl] }];
        },
        collectResults: (state, results) => ({
          ...state,
          data: { ...state.data, stitchedUrl: results[0]?.videoUrls[0] },
        }),
      },
    ],
  };

  try {
    const finalState = await executePipeline(config, context, callbacks);

    const stitchedUrl = finalState.data.stitchedUrl as string | undefined;
    if (!stitchedUrl) {
      return JSON.stringify({
        error: 'stitch_failed',
        message: 'Orbit completed angle and video generation but stitching failed.',
      });
    }

    // Present all results: angle images + stitched video + individual clips
    callbacks.onToolComplete(
      'orbit_video',
      finalState.imageUrls,
      [stitchedUrl, ...finalState.videoUrls],
    );

    return JSON.stringify({
      success: true,
      resultCount: 1,
      mediaType: 'video',
      creditsCost: formatCredits(totalCost),
      message: `Successfully created 360° orbit video (4 angles, 4 transitions, 1 stitched). Cost: ~${formatCredits(totalCost)} credits.`,
    });
  } catch (err) {
    const message = err instanceof Error ? err.message : String(err);
    console.error('[ORBIT] Pipeline failed:', message);
    return JSON.stringify({ error: 'orbit_failed', message });
  }
}
```
